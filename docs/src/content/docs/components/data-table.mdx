## Basic

<Example
  code={`import { DataTable } from '@eekodigital/raster/data-table';\nimport type { DataTableColumnDef } from '@eekodigital/raster/data-table';\n\ntype Row = { id: string; name: string; status: string };\n\nconst columns: DataTableColumnDef<Row>[] = [\n  { accessorKey: 'ref', header: 'Ref' },\n  { accessorKey: 'name', header: 'Name' },\n  { accessorKey: 'status', header: 'Status' },\n];\n\n<DataTable columns={columns} data={rows} />`}
>
  <DS_DataTable client:load columns={columns} data={rows} />
</Example>

## Multi-column sorting

All columns are sortable by default. Click a header to sort ascending, click again for
descending, click a third time to clear. Hold <kbd>Shift</kbd> to sort by multiple columns
simultaneously.

<Example
  code={`// All columns are sortable — no extra config needed.\n// Shift+click a second header to add a secondary sort.\n<DataTable columns={columns} data={rows} />`}
>
  <DS_DataTable client:load columns={columns} data={rows} />
</Example>

## Pagination

Pass `pageSize` to enable built-in pagination. Previous/next controls appear
below the table when the data exceeds one page.

<Example code={`<DataTable columns={columns} data={rows} pageSize={3} />`}>
  <DS_DataTable client:load columns={columns} data={rows} pageSize={3} />
</Example>

## Filtering

Add `filter` to show a search input above the table that filters across all
column values simultaneously. Use `filterPlaceholder` to customise the input text.

<Example code={`<DataTable columns={columns} data={rows} filter />`}>
  <DS_DataTable client:load columns={columns} data={rows} filter />
</Example>

## Column resizing

Add `resizable` to enable drag handles on column borders. Individual columns can
opt out by setting `enableResizing: false` in their column definition.

<Example code={`<DataTable columns={columns} data={rows} resizable />`}>
  <DS_DataTable client:load columns={columns} data={rows} resizable />
</Example>

## Column pinning

Pass `pinnedColumns` to keep columns sticky at the left or right edge while the
table scrolls horizontally. Use column `id` or `accessorKey` to
identify which columns to pin. Set explicit `size` values in your column
definitions for accurate sticky offsets when multiple columns are pinned.

<Example
  code={`<DataTable\n  columns={columns}\n  data={rows}\n  pinnedColumns={{ left: ['ref'], right: ['status'] }}\n/>`}
>
  <DS_DataTable
    client:load
    columns={wideColumns}
    data={wideRows}
    pinnedColumns={{ left: ["ref"], right: ["status"] }}
  />
</Example>
