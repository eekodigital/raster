## Setup

<div class="callout">
  <p>
    Add <code>Toast.Provider</code> and <code>Toast.Viewport</code> once in your app root. The
    viewport is the fixed portal that renders toasts — position it after your main content.
  </p>
</div>

<Example
  code={`// app/root.tsx or layout component\nimport { Toast } from '@eekodigital/raster';\n\nexport function AppLayout() {\n  return (\n    <Toast.Provider swipeDirection="right">\n      <App />\n      <Toast.Viewport />\n    </Toast.Provider>\n  );\n}`}

## When to use

<div class="callout">
  <p>
    <strong>Use Toast for</strong> transient, non-blocking feedback — "Report saved", "Link copied",
    "Changes applied". The user doesn't need to act; the message auto-dismisses.
  </p>
  <p>
    <strong>Don't use Toast for</strong> errors that require action. Use inline validation for form
    errors, or <a href="/components/alert-dialog">AlertDialog</a> for destructive confirmations.
  </p>
  <p>
    <strong>Don't use Toast for</strong> information the user must not miss. Toasts can be dismissed
    before they're read — use a persistent banner or callout instead.
  </p>
</div>
